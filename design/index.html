Brand story

Three ideas, braided into one mark.

Rainy is Nordic, calm and a little magical — and it's a serious pro tool underneath. The brand lives where those meet. Everything traces back to three image-systems pulled straight from the moodboard.

🌧️

Rain = making it rain

For a creator, "making it rain" is reach, views, growth. Rainy turns the work of research and ideation into downpour: the agent's job is abundance. Raindrops, umbrellas, weather — the optimistic side of the brand.

🦌

Antlers = branching ideas

The killer metaphor. A reindeer's antlers branch exactly like a mindmap. Every idea on the canvas grows tines: a video splits into hooks, hooks into angles, angles into scripts. The canvas literally renders connections as antler branches.

❄️

Nordic = clean & calm

Snow, aurora, pine, reindeer. A Scandinavian design ethic: brutal clarity, generous whitespace, quiet confidence. The aurora gradient is our one permitted flourish — used like a spark, never wallpaper.

The creator workflow we're designing for

From the Sunstead map — the canvas + Home together have to carry all three stages without ever feeling like three apps.

STAGE 1
Understand & Setup

Link creators & channels, own + competitors. Build a living personal context. "How does the creator look?"

STAGE 2
Research & Ideate

Synthesis, scripting, storyboards, shotlists. Custom skills + templates. Agent spins up artifacts on demand — this is the canvas.

STAGE 3
Performance & Analytics

Insights, coaching. Analyze own + competitor channels, niche, trends. Learn patterns & creative opportunity gaps — this is Home.

Pairing rules

  • ◦ Never mix Fraunces + Inter in the same headline.
  • ◦ Numerals are always Inter or Mono — tabular figures on in tables (font-variant-numeric: tabular-nums).
  • ◦ Body line-length capped at ~68 chars.
  • ◦ UPPERCASE only for 10–12px eyebrows, +8% tracking.
  • ◦ Min UI text 11px; min interactive label 12px.

Principle · Progressive disclosure

Brutally simple at the door. A pro tool once you're in.

Like Figma: a newcomer sees one canvas and one input. Every additional control earns its place by being summoned — by a selection, a hover, a keystroke, or the agent. Nothing is hidden that you need; nothing is shown that you don't.

LEVEL 0
Empty
One input. The whole UI is "paste a link."
LEVEL 1
Seeded
Nodes appear. Tool rail + zoom fade in. Select → contextual bar.
LEVEL 2
Agent live
Rainey branches ideas (antlers), drops glowing nodes, narrates in a toast.
LEVEL 3
Power user
Layers + inspector + pages + multiplayer. Earned, never forced.

What summons each layer

  • Hover a node → resize handles + quick-connect dots.
  • Select a node → floating contextual toolbar (the "selbar").
  • ⌘K → command palette: skills, templates, "ask Rainey".
  • Right-edge hover → inspector slides in.
  • First share → multiplayer avatars + comments appear.
  • Agent action → toast + glow + antler branches.

Rules we hold the line on

  • ◦ Default canvas chrome ≤ 3 visible controls (doc title, zoom, Rainey).
  • ◦ Panels remember per-user state but always start collapsed for a new project.
  • ◦ The agent never opens a panel for you — it works in the canvas.
  • ◦ Every advanced action has a plain-language Rainey equivalent.
  • ◦ No modal dialogs on the canvas. Ever. Inline or it doesn't exist.
